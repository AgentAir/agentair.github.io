最初につまずいたポイント

1. VNetは「クラウド上のL3ネットワーク」として考える

Azure Virtual Network、いわゆるVNetは、オンプレミスでいう物理スイッチやルータそのものではなく、 Azure上で論理的に分離されたネットワーク空間として理解すると整理しやすいです。

2. SubnetはIPアドレス範囲の分割だけではない

Subnetは単なるアドレス範囲の分割ではなく、NSG、ルートテーブル、Private Endpointなどの設計単位にもなります。 オンプレミスのVLANに近い感覚で見つつ、Azure固有の制御がある点に注意が必要です。

3. NSGはACLに似ているが、完全に同じではない

NSGは通信の許可・拒否を制御する点ではACLに近いですが、サブネットまたはNICに関連付ける点、 優先度で評価される点、既定ルールが存在する点など、オンプレミス機器のACLとは運用感が異なります。

4. Azure Bastionは踏み台サーバとは少し違う

Azure Bastionは、VMにパブリックIPを付けずにブラウザ経由でRDP/SSH接続するためのマネージドサービスです。 自分でLinux踏み台を立てる構成とは、管理範囲やセキュリティ設計の考え方が異なります。

5. Private Endpointは名前解決まで含めて理解する必要がある

Private Endpointは、Azure PaaSサービスへプライベートIPで接続するための仕組みですが、 実際にはDNSの名前解決が重要になります。 IPだけ見ていると理解しにくく、FQDNがプライベートIPへ解決される流れまで確認する必要があります。
