Claude CodeをWindowsに入れてGitHub Pagesブログを更新するまで

2026-04-30 / Claude Code / GitHub / Windows

Home Articles About

はじめに

このブログはGitHub Pagesで公開している手書きHTML/CSSの個人ブログです。 記事を追加するたびに3つのファイル（記事HTML・articles.html・index.html）を手作業で編集していましたが、 Claude Codeを使えばこの作業を効率化できると考えました。

今回は実家のPCで作業したため、Claude Code自体のインストールから始める必要がありました。 本記事は、その導入から実際にブログを更新してGitHubへpushするまでの流れと、途中で詰まった点を整理した個人の作業メモです。

実施した流れ

1. Claude Codeのインストール

Claude CodeはNode.jsのnpmパッケージとして提供されています。 インストールはターミナルから以下のコマンドで行いました。 

npm install -g @anthropic-ai/claude-code

インストール自体はエラーなく完了しましたが、この後すぐに問題が発生しました。

2. PATH への追加

インストール後に claude コマンドを実行しようとしたところ、認識されませんでした。 確認すると、npmのグローバルインストール先（C:\Users\<ユーザー名>\.local\bin）がPATHに含まれていないことが原因でした。

Windowsの「システムの詳細設定」からユーザー環境変数のPATHに該当パスを追加し、 ターミナルを再起動することで claude コマンドが認識されるようになりました。

3. GitHubリポジトリのclone

ブログのリポジトリをローカルにcloneしました。 cloneしたフォルダ（agentair.github.io）の中に移動してから、Claude Codeを起動しました。 起動時にリポジトリのルートにいることを確認してから操作を始めることが重要でした。

4. Claude Codeでの作業

Claude Codeにリポジトリの構成を確認させ、既存の記事ページのHTML構造を把握させました。 その後、記事のタイトル・タグ・概要・本文の要件を指示することで、 記事HTML・articles.html・index.htmlの3ファイルをまとめて編集させました。

5. 差分確認とpush

Claude Codeが生成したファイルを git diff で確認し、意図しない変更や機密情報が含まれていないかチェックしました。 問題がないことを確認した後、git add / git commit / git push でGitHubへ反映しました。

詰まったポイント

claude コマンドが認識されなかった

npmでグローバルインストールしても claude コマンドが使えない場合、 インストール先のパスがPATHに含まれていないことが原因です。 Windowsの場合、C:\Users\<ユーザー名>\.local\bin をユーザー環境変数のPATHに追加し、 ターミナルを再起動することで解決しました。

fatal: not a git repository が出た

Gitコマンドをリポジトリのフォルダ外で実行したため、fatal: not a git repository というエラーが出ました。 pwd コマンドで現在地を確認し、cloneしたフォルダの中にいることを確認してから再実行して解決しました。 作業前に現在のディレクトリを確認する習慣が重要だと改めて感じました。

Author identity unknown が出た

git commit を実行したところ、Author identity unknown というエラーが出てコミットできませんでした。 初めて使うPCでGitのユーザー情報が設定されていないことが原因でした。 以下のコマンドでグローバル設定を行い、解決しました。 

git config --global user.name "Your Name"
git config --global user.email "your@example.com"

git push 時にGitHub認証が必要だった

git push を実行したところ、GitHub認証を求めるフローが走りました。 GitHubはパスワード認証を廃止しているため、Personal Access Token（PAT）またはGitHub CLIによる認証が必要です。 認証情報はコードやファイルに直接書かず、認証フローを通じて設定しました。

学んだこと

Claude Codeが便利でもGitの基本は必要

Claude Codeがファイルの生成・編集を自動化してくれても、 git add / git commit / git push の一連の操作や、 エラーの意味を理解するためのGit基礎知識は必要です。 ツールに任せる部分と自分で理解しておく部分を分けて考えることが重要です。

作業前にpwdで現在地を確認する

Gitコマンドはリポジトリのフォルダ内で実行する必要があります。 ターミナルを開いたときにどこにいるかを意識していないと、意図しないディレクトリでコマンドを実行してエラーになります。 特に複数の作業を同時に行うときは、pwd で現在地を確認する習慣をつけました。

Claude Codeが作った内容もgit diffで確認する

Claude Codeが生成したファイルだからといって、そのままcommitするのではなく、 git diff で変更内容を自分の目で確認することが大切です。 意図しない変更が混入していないか、不要な内容が含まれていないかをチェックしてからcommitするようにしました。

公開前に機密情報が入っていないか確認する

GitHub Pagesは公開リポジトリであるため、pushした内容はそのままインターネット上に公開されます。 会社情報、顧客情報、実IPアドレス、認証情報、SAS URLなどが含まれていないかを push前に必ず確認することを原則としています。

記事追加のたびに編集するファイルは3つある

このブログに記事を追加する場合、必ず編集が必要なファイルは次の3つです。

  • articles/<新記事>.html（記事本文：新規作成）
  • articles.html（記事一覧：先頭にカード追加）
  • index.html（トップページの最新記事：先頭にカード追加）

この3ファイルの編集をClaude Codeにまとめて依頼することで、手作業より効率よく更新できるようになりました。

今後の運用方針

Claude Codeへの指示を明確にする

Claude Codeに記事を作らせる際は、タイトル・タグ・概要・含めたい内容・禁止事項をセットで指示するようにしました。 あいまいな指示だと、意図しない内容が含まれる可能性があるためです。

生成されたHTMLを確認してからcommitする

Claude Codeが生成したHTMLは、commitする前に git diff で内容を確認します。 特に、リンクのパス（../ が必要な部分など）や機密情報の混入がないかを重点的にチェックします。

実際に検証してから記事にする

Terraformのようにまだ実機で試していない技術については、概念整理の記事は書けますが、 検証メモや手順メモは実際に手を動かしてから書くようにします。 実際に詰まった点こそが、将来の自分や同じ立場の読者に役立つ内容だと考えているためです。

まとめ

Claude Codeを使うことで、GitHub Pagesブログの記事追加作業を大幅に効率化できました。 ただし、導入時のPATH設定、Gitのユーザー設定、GitHub認証といったセットアップ手順は 自分で理解して対処する必要があります。

ツールに頼りすぎず、生成された内容を自分の目で確認してからpushするフローを維持することが、 品質と安全性を保つために重要だと感じました。
