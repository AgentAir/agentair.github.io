別PCでCodex CLIを使ってGitHub Pagesブログを更新できるようにする手順

2026-04-30 / Codex / GitHub Pages / Windows

Home Articles About

はじめに

実家PCでCodex CLIを導入し、GitHub Pagesブログの更新に使えることを確認しました。 次に自宅PCや別PCでも同じ手順を再現できるように、作業ログとして手順をまとめます。

目的は、どのPCでもブログ記事の追加や修正ができる状態にすることです。 この記事は公式ドキュメントの代替ではなく、自分用の再現手順として整理したものです。

前提

  • Windows PCを想定します。
  • GitHub Pagesブログのリポジトリは agentair.github.io です。
  • GitHubアカウントにログインできる状態を前提にします。
  • ChatGPTアカウントでCodexを利用できる状態を前提にします。
  • 公開リポジトリなので、機密情報は置きません。

会社情報、顧客情報、実IPアドレス、認証情報、SAS URL、試験問題本文などは扱いません。 Windowsユーザー名も実名ではなく、%USERPROFILE% や <username> のように抽象化して扱います。

Node.js / npm の確認

まず、Node.jsとnpmが使えるか確認します。 

node -v
npm.cmd -v

node が入っていなければ、Node.js LTSをインストールします。 PowerShellでは npm が実行ポリシーによりブロックされる場合があるため、ここでは npm.cmd を使います。

Codex CLI のインストール

npmからCodex CLIをインストールします。 

npm.cmd install -g @openai/codex

インストール後、バージョンとコマンドの場所を確認します。 

codex --version
where.exe codex

GitHubブログリポジトリをcloneする

ブログ用のリポジトリをローカルにcloneします。 コマンドプロンプト寄りの書き方では、次のように進めます。 

mkdir "%USERPROFILE%\Documents\GitHub"
cd "%USERPROFILE%\Documents\GitHub"
git clone https://github.com/AgentAir/agentair.github.io.git
cd agentair.github.io
git status

PowerShellの場合は、次の書き方でも進められます。 

mkdir "$env:USERPROFILE\Documents\GitHub"
cd "$env:USERPROFILE\Documents\GitHub"
git clone https://github.com/AgentAir/agentair.github.io.git
cd agentair.github.io
git status

Codexをブログフォルダで起動する

Codexは必ずブログリポジトリの中で起動します。 起動前に git status で working tree clean を確認しておきます。 

cd "$env:USERPROFILE\Documents\GitHub\agentair.github.io"
git status
codex

Codexに最初に渡す確認プロンプト

最初はすぐ編集させず、リポジトリ構成を確認してもらうところから始めます。 

このリポジトリは、GitHub Pagesで公開している手書きHTML/CSSの個人ブログです。

目的：
- 既存のHTML構造を大きく変えずに保守する
- 記事追加やリンク修正を効率化する
- 会社情報、顧客情報、実IP、認証情報、SAS URL、試験問題本文は含めない
- 変更前に編集対象ファイルと変更方針を提示する

まず、このリポジトリの構成を確認し、Home / Articles / About / articles配下の記事の役割を整理してください。
まだファイルは編集しないでください。

作業後の確認

Codexが作業した後は、必ず差分を確認します。 

git status
git diff
  • Codexが意図しないファイルを変更していないか確認します。
  • 会社情報、実IP、認証情報、SAS URL、試験問題本文が含まれていないか確認します。
  • 記事ページ、一覧ページ、トップページのリンクパスが正しいか確認します。

commit / push

差分に問題がなければ、commitしてGitHubへpushします。 

git add .
git diff --cached
git commit -m "任意のコミットメッセージ"
git push

よくある詰まりポイント

npm がPowerShellの実行ポリシーでブロックされる

npm が実行できない場合は、npm.cmd を使います。

fatal: not a git repository が出る

git status をリポジトリ外で実行している可能性があります。 agentair.github.io フォルダへ移動してから再実行します。

Author identity unknown が出る

初めて使うPCでは、Gitのユーザー名とメールアドレスが未設定の場合があります。 

git config --global user.name "Your Name"
git config --global user.email "your@example.com"

Codexを違うフォルダで起動してしまう

pwd で現在地を確認し、ブログリポジトリの中に移動してから codex を起動します。

注意点

  • ChatGPT履歴JSONやOneNote生データを公開リポジトリに置きません。
  • 公開ブログに載せるのは、整理済み・マスク済みの内容だけにします。
  • Codexの出力も必ず git diff で確認します。
  • 装飾画像ではなく、必要な場合は技術理解を助ける構成図を使います。

まとめ

Codex CLIを使えば、別PCでもGitHub Pagesブログを更新できる状態を作れます。 重要なのは、Codexを入れることだけではなく、git diff で確認して安全にpushする運用を作ることです。

自宅PCでも同じ手順で環境を再現できるように、Node.js、Codex CLI、clone、Codex起動、commit / pushまでの流れを残しました。 今後もPCを変えて作業する場合は、この手順を起点に確認します。
