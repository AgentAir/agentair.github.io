GitHub Pagesで手書きHTML/CSSブログを公開するまで

2026-04-30 / GitHub / GitHub Pages / HTML / CSS

Home Articles About

はじめに

個人ブログを作成するにあたり、まずはGitHub Pagesを使って、 手書きのHTML/CSSサイトを公開しました。

最初からAstroやHugoなどの静的サイトジェネレーターを使う方法もありますが、 今回は「まず公開すること」を優先し、最小構成で始めました。

今回のゴール

今回のゴールは、凝ったブログシステムを作ることではなく、 インターネット上に自分の技術メモを置ける場所を作ることです。 

GitHubリポジトリ
  ↓
index.html / css/style.css
  ↓
GitHub Pages
  ↓
https://ユーザー名.github.io/

作成した構成

最初は以下のようなシンプルな構成にしました。 

agentair.github.io/
├── index.html
├── about.html
├── articles.html
├── css/
│   └── style.css
└── articles/
    ├── 2026-04-azure-network-first-step.html
    ├── 2026-04-azure-private-endpoint-storage.html
    └── 2026-04-github-pages-static-blog.html

HTMLを手書きすることで、ページ構造やリンクの関係を理解しやすくなります。 一方で、記事数が増えると手動更新が大変になるため、将来的にはAstroなどへの移行も検討できます。

作業手順

1. GitHubでリポジトリを作成

GitHub上で、ユーザーサイト用のリポジトリを作成しました。 GitHub Pagesでユーザーサイトとして公開する場合、リポジトリ名は以下の形式にします。 

<GitHubユーザー名>.github.io

今回は以下のようなリポジトリ名にしました。 

agentair.github.io

2. index.htmlを作成

まずはトップページとなるindex.htmlを作成しました。 GitHub Pagesでは、リポジトリ直下のindex.htmlがトップページとして表示されます。

3. CSSを追加

次に、css/style.cssを作成し、見た目を整えました。 最初はデザインに凝りすぎず、読みやすさを優先しました。

4. AboutページとArticlesページを作成

ブログとして最低限必要なページとして、Aboutページと記事一覧ページを作成しました。

5. 記事ページを作成

articlesフォルダを作成し、記事ごとにHTMLファイルを配置しました。 各記事ページからHome、Articles、Aboutへ戻れるようにナビゲーションも設定しました。

詰まったポイント

1. リポジトリ名と公開URLの関係

GitHub Pagesのユーザーサイトでは、リポジトリ名とGitHubユーザー名の関係が重要です。 ユーザー名と異なるリポジトリ名にすると、想定していたURLで表示されず、404になることがあります。

例えば、GitHubユーザー名がAgentAirの場合、ユーザーサイト用のリポジトリ名は以下になります。 

agentair.github.io

この場合、公開URLは以下になります。 

https://agentair.github.io/

2. 反映に少し時間がかかる

GitHub Pagesの設定後、すぐには表示されず404になることがあります。 その場合は、数分待ってから再読み込みすると表示されることがあります。

3. 相対パスに注意する

トップページと記事ページでは、CSSへのパスが異なります。

トップページの場合：

<link rel="stylesheet" href="css/style.css">

articlesフォルダ内の記事ページの場合：

<link rel="stylesheet" href="../css/style.css">

記事ページは1階層下にあるため、../ を付けて1つ上の階層に戻る必要があります。

今回の学び

  • 最初から完璧なブログ基盤を作ろうとしなくてよい
  • GitHub PagesならHTML/CSSだけで公開できる
  • ユーザーサイトではリポジトリ名が重要
  • まず公開してから、少しずつ改善する方が進みやすい
  • 記事が増えてきたら、AstroやHugoなどへの移行を考えればよい

今後やりたいこと

  • 記事を増やす
  • カテゴリごとに整理する
  • コードブロックを見やすくする
  • スマートフォン表示を調整する
  • 記事数が増えたらAstroなどへの移行を検討する

まとめ

GitHub Pagesを使うことで、手書きHTML/CSSだけでも個人ブログを公開できます。 最初から高度な構成にする必要はなく、まずはindex.html、style.css、記事ページを作るだけで十分です。

今回のように、実際に作業した内容を記事として残すことで、 回り道も技術ナレッジとして資産化できます。
