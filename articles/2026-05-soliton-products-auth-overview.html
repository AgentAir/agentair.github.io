Soliton製品群の役割整理：FileZen S / SecureBrowser / SmartOn / OneGate / NetAttest EPS の違い

2026-05-13 / Network Security / Authentication / Study Log

Home Articles About

はじめに

Soliton製品を調べていると、認証、Web分離、ファイル受け渡し、ネットワーク認証など、似た言葉が何度も出てきます。 名前だけを追うと、FileZen S、Soliton SecureBrowser、Soliton SecureGateway、SmartOn、OneGate、NetAttest EPS が同じ「セキュリティ製品群」に見えてしまい、どこで使うものなのかが混ざりやすいと感じました。

この記事では、単なる製品紹介ではなく、ChatGPTとの対話で自分の理解がどう変わったかを残します。 公式仕様や最新の対応範囲は導入時に必ず確認するとして、ここではネットワークエンジニア目線で「何に近いか」「何ではないか」「最後にどう理解したか」を整理します。

まず全体像

最初に、自分の中で次のように置き場所を分けると理解しやすくなりました。 認証という言葉が出てきても、Windowsログオン、SaaSログイン、Wi-Fi/VPN接続認証では主戦場が違います。

FileZen S：FTPサーバか、ファイルサーバか、ファイル受け渡しゲートウェイか

最初の理解

最初は「ファイルを置いて、別の場所から取る」という動きだけを見て、FTPサーバのようなものかと思いました。 あるいは、社内ファイルサーバに承認やログを足したもの、という理解にも寄りました。

途中で修正した点

ただ、自治体NWやインターネット分離の文脈で見ると、普通のファイル共有とは目的が違います。 重要なのは、分離されたネットワーク間でファイルを移すときに、承認、ウイルスチェック、無害化連携、ログ、保存期間管理を入れられることです。 USBメモリで手渡しする運用を、Web画面とポリシーで置き換えるものに近いと考えると整理しやすくなりました。

最終理解

FileZen Sは、FTPサーバでも通常のファイルサーバでもなく、分離ネットワーク間のファイル受け渡しゲートウェイとして理解するのがよさそうです。 ファイルを保管し続ける場所というより、「安全に渡す」「必要な確認を通す」「履歴を残す」ための製品として見ると、役割がはっきりします。

SecureBrowser / SecureGateway：Webアプリか、専用ブラウザか、ゲートウェイは何か

最初の理解

SecureBrowserという名前だけを見ると、ブラウザでアクセスするWebアプリケーションのようにも見えました。 しかし実際の理解としては、端末に入れて使う専用ブラウザアプリと捉える方が自然でした。

途中で修正した点

SecureGatewayについても、最初はネットワーク機器、ルータ、プロキシのようなものなのかと考えました。 途中で整理すると、SecureBrowserと組み合わせて使うゲートウェイサーバー、または物理/仮想アプライアンスという見方がしっくりきました。 NW機器というより、DMZなどに置いてアクセス先や通信を制御する中継点です。

最終理解

SecureBrowserは、端末側で動かすセキュアブラウザです。 ダウンロード、コピー、保存などを制御し、端末に業務データを残しにくくする考え方です。 SecureGatewayは、そのブラウザの先にある中継・制御ポイントとして理解しました。

VDIや画面転送そのものではなく、端末側で専用ブラウザを起動してWebアクセスを制御する、という違いを押さえると混同しにくくなります。

SmartOn：RADIUSサーバではなくWindowsログオンMFA

最初の理解

SmartOnは多要素認証を行う製品なので、最初は「つまりRADIUSサーバなのか」と考えました。 MFA、証明書、認証サーバという言葉が出てくると、ネットワークエンジニアとしてはRADIUSに寄せて理解したくなります。

途中で修正した点

ここで一度、RADIUSという大きな枠の中にNetAttest EPSがあり、その中にSmartOnがあるような誤解をしそうになりました。 しかしSmartOnの中心は、Wi-FiやVPNの接続認証ではなく、Windows PCにログオンするときの本人確認です。

最終理解

SmartOnはRADIUS系の製品ではなく、WindowsログオンMFAの製品として理解します。 ICカード、顔認証、指紋、指静脈などで「このPCを使い始める人が本人か」を確認するものです。 サーバーソフト、クライアントソフト、PC台数ライセンス、認証デバイスやトークンのライセンスを組み合わせて導入するイメージです。

RADIUS系の基本担当は、Soliton製品群の中ではNetAttest EPSです。 SmartOnとNetAttest EPSはどちらも認証に関係しますが、入口が「PCログオン」なのか「ネットワーク接続」なのかが違います。

OneGate：RADIUSサーバか、CA/SSO/MFA基盤か

最初の理解

OneGateはクラウド型で、MFAや証明書、SSOに関係するため、最初は「クラウド版のRADIUS系製品なのか」と考えました。 さらに、ユーザーや認証を扱うならADサーバのようなものなのか、という方向にも理解が揺れました。

途中で修正した点

整理していくと、OneGateはRADIUSサーバそのもの、ADそのものというより、ADやEntra IDなどと連携しながら使うクラウド型の認証基盤として見た方が自然でした。 CA、MFA、SSO、ID認証をまとめる入口であり、利用者がSaaSやWebサービスにログインするときの基盤です。

最終理解

OneGateはクラウド型のCA/MFA/SSO/ID認証基盤です。 Wi-FiやVPN認証にも関係しますが、RADIUSの現地受け口はNetAttest EPS-edgeなどと連携する構成として理解するのがよさそうです。 つまり、OneGateだけを「RADIUSサーバ」と呼ぶと少し雑になります。

SmartOnと機能が一部重なって見えるのは、どちらもMFAや証明書、ログイン制御の言葉を持つからです。 ただしSmartOnはWindowsログオン、OneGateはSaaS/クラウド/Webログインが主戦場です。

SmartOnとOneGateはなぜ機能がかぶって見えるのか

この2つは、最も混同しやすい組み合わせでした。 どちらも認証製品で、MFA、証明書、ログイン制御という用語が出てきます。 しかし、使う場面を分けると見通しがよくなります。

観点 SmartOn OneGate
中心領域 Windows PCログオン SaaS、クラウド、Webログイン
利用者の動き PCを使い始める 業務サービスへログインする
近いもの PCログオン強化、端末本人確認 IDaaS、SSO、クラウド認証基盤
混同しやすい理由 MFAや証明書を使う MFAや証明書を使う

最終的には、「SmartOnはPCの入口」「OneGateはクラウドサービスの入口」と置くと、自分の中では迷いにくくなりました。

NetAttest EPS：基本的なRADIUS系を担当

NetAttest EPSは、ネットワークエンジニアにとって一番イメージしやすい製品でした。 Wi-Fi、VPN、有線LANなどの接続時に、AP、無線LANコントローラ、スイッチ、VPN装置からRADIUS認証要求を受ける側です。

ここではRADIUS、802.1X、EAP-TLS/EAP-PEAP、プライベートCA、証明書認証といった言葉が中心になります。 「この端末・このユーザーをネットワークに入れてよいか」を判断する基盤です。

まとめ

今回の整理で一番大きかったのは、「認証」という言葉を一つにまとめないことでした。 Windowsログオン、SaaSログイン、Wi-Fi/VPN接続認証は、同じ認証でも入口も関係する機器も違います。

FileZen Sは分離ネットワーク間のファイル受け渡し、SecureBrowser/SecureGatewayはWebアクセス制御、SmartOnはWindowsログオンMFA、OneGateはクラウド型のCA/MFA/SSO/ID認証基盤、NetAttest EPSはRADIUS/802.1X系のネットワーク認証基盤。 こう置き直すと、自治体NW、インターネット分離、認証基盤、RADIUS、MFA、SSO、ファイル受け渡しの文脈がかなり整理しやすくなりました。

注意: この記事は学習ログとしての整理です。製品仕様、ライセンス、構成可否、対応OS、対応認証方式は更新されるため、実案件では必ず公式資料とベンダー確認を前提にします。
