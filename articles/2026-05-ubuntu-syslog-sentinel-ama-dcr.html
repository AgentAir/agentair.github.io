Ubuntu の Syslog を Azure Arc / AMA / DCR 経由で Microsoft Sentinel に収集する手順

Azure Arc に接続しただけでは Syslog は収集されない。AMA と DCR の関連付けまで含めて確認した学習メモです。

最初の理解

最初は、Ubuntu が Azure Arc に接続されていれば、そのまま Microsoft Sentinel にログが送られると思っていました。 しかし実際には、Azure Arc に接続しただけでは不十分でした。

Azure Arc は、オンプレミスや他クラウド、通常の Ubuntu などを Azure 上の管理対象リソースとして扱うための仕組みです。 ただし、ログを実際に収集して Log Analytics Workspace に送るには、Azure Monitor Agent（AMA）と Data Collection Rule（DCR）の設定が必要です。

実際に必要だったもの

  • Azure Arc 対応サーバーとして Azure に登録されていること
  • Azure Monitor Agent / AMA が導入されていること
  • Data Collection Rule / DCR が作成されていること
  • DCR が対象の Ubuntu / Arc 対応サーバーに関連付けられていること
  • DCR で Syslog の Facility / Severity を収集対象にしていること
  • DCR の送信先が Sentinel で使っている Log Analytics Workspace になっていること

DCR 関連付けの確認

Ubuntu 側では、AMA の設定配布状況を以下のディレクトリで確認しました。 

sudo ls -l /etc/opt/microsoft/azuremonitoragent/config-cache/configchunks/

最初は configchunks に設定が配布されていませんでした。 その後、DCR 作成時に Azure Arc 対応サーバーを選択し、対象 Ubuntu に関連付けたところ、 configchunks 配下にファイルが増えました。

このことから、DCR が対象 Ubuntu に関連付けられると、AMA 側に収集設定が配布されることが確認できました。

Heartbeat の確認

以下の KQL では、Windows と Ubuntu の両方が見えていました。 

Heartbeat
| where TimeGenerated > ago(1h)
| where Category == "Azure Monitor Agent"
| order by TimeGenerated desc

これは、AMA 自体が動作して Log Analytics Workspace に接続できていることの確認になります。 ただし、Heartbeat が見えていることは「AMA が接続できている」ことの確認であり、 「Syslog が収集できている」ことの確認ではありません。

Syslog の確認

Syslog の収集確認には、以下の KQL を使用しました。 

Syslog
| where TimeGenerated > ago(1h)
| where SyslogMessage contains "sentinel-test"
| project TimeGenerated, Computer, HostName, Facility, SeverityLevel, ProcessName, SyslogMessage
| order by TimeGenerated desc

最初は sentinel-test が見えませんでした。 しかし、時間を置いてから再度確認すると、Syslog テーブルに sentinel-test が表示されました。

そのため、DCR 関連付け後すぐに見えない場合でも、少し時間を置いてから再確認する必要があります。

最終的な理解

  • Azure Arc は、Ubuntu を Azure 上の管理対象リソースとして扱うための仕組み
  • AMA は、実際にログを収集して Azure Monitor / Log Analytics に送るエージェント
  • DCR は、どのログを、どの条件で、どの Log Analytics Workspace に送るかを決めるルール
  • Microsoft Sentinel は、Log Analytics Workspace に入ったログを使って分析・検知・調査を行うサービス

今回の一番大きな学びは、Arc に接続しただけでは Syslog は収集されないという点です。 Heartbeat が見えていても、それだけでは Syslog 収集成功とは言えません。 最終的には Syslog テーブルで実際のログを確認する必要があります。

まとめ

Ubuntu から Microsoft Sentinel に Syslog を収集するには、Azure Arc、AMA、DCR、Log Analytics Workspace の関係を理解する必要があります。 Arc は管理対象として登録するための入口であり、AMA がログを送り、DCR が収集ルールを定義し、Sentinel が Log Analytics Workspace 上のログを分析します。
