AZ-104の学習計画を立て直したメモ（教科書先行・問題集集中型）

2026-06-03 / Azure / Study

Home Articles About

はじめに

AZ-104（Microsoft Azure Administrator）の学習計画を、自分の実際のペースに合わせて立て直したときのメモです。後から「どういう計画で進めることにしたか」を思い出せるように、要点だけを残しておきます。

この記事は学習の進め方を整理した個人の運用メモです。試験問題の本文や選択肢、問題集の転載は含みません。また公式ドキュメントの代替を意図したものではありません。

前提

  • 教材：Microsoft認定資格試験テキスト AZ-104（教科書）＋ 問題集687問
  • 教科書は全8章構成。2-3章まで実施済みで、残りは2-4以降〜第7章（残り約482ページ）
  • 第8章（模擬試験）は実施しない
  • 受験目標：できるだけ早く。目安は1.5〜2ヶ月以内
  • TOEIC学習は別レーンで並行（学習計画と進捗を連動させない）

計画の考え方

最初は「教科書を章ごとにまとめて、その章の問題を解く」流れを考えていましたが、使っている問題集が単元ごとに分類しづらい構成だったため、教科書を先に全部終わらせてから問題集をまとめて回す方針に変えました。

また、AZ-104は知識を問う以上に「この状況でどの設定・サービスが正解か」を問う比重が高いと判断し、問題演習の時間を厚めに確保する構成にしています。

フェーズ構成

フェーズ1：教科書を全節まとめる（約1.5週）

  • 2-4 Azure CLI 〜 第7章コスト管理までを、節（2-xなど）単位で区切ってOneNoteにまとめる
  • ページ数ではなく節単位で区切ることで、途中で中途半端に終わらないようにする
  • 本業（ネットワーク）と近い第6章ネットワークサービスは進みが速いので飛ばし気味に、本業と距離のあるEntra IDやストレージのアクセス制御などに時間を残す
  • まとめは全文写経ではなく要点抽出に徹する（数値・制約・各サービスの役割）

教科書フェーズの1日の進め方

区切りは「ページ」ではなく「節」で考える。「今日はこの節まで」で終えれば途中で切れない。節の重さに応じて1日の節数を増減するだけでよい。

区分 1日の目安
平日 2〜3節
休日 4〜5節

この量で約2週（全41節）で教科書を読了する。

ただし以下の重い節は、その日は「1節で1日」と割り切る。

重い節 ページ数
3-2 Microsoft Entra ID 40ページ
4-3 Azure Blob Storage 43ページ

逆に監視系（7-3〜7-7）は1節2〜5ページの軽い節が並ぶので、1日でまとめて進められる。

フェーズ2：問題集687問を1周（1日30問・約23日）

  • 1日30問ペース（正味60〜90分）で687問を1周する
  • 間違えた問題には印をつける
  • 解説を読んだら、対応するOneNoteのページに戻って加筆する（問題集が単元対応していなくても、自分でテーマに紐づければ教科書と問題集がつながる）

フェーズ3：弱点2周目＋受験（約1週）

  • 印をつけた問題だけを再演習する（同じく30問/日ペース）
  • 正解済みの問題は捨てて時間を節約する
  • 安定して解けるようになったら受験を予約する

予備：減速の吸収帯（残り約2週）

  • 忙しい週や詰まった箇所の遅れをここで吸収する
  • 順調なら使わずに前倒し受験してよい

ペースの根拠

問題集687問を、1周＋誤答分（仮に3割＝約206問）の2周目として、1日あたりの問題数別に所要日数を見積もった。

1日の問題数 問題集フェーズ 教科書込みの全体所要
15問 約8.6週 約10週（2.5ヶ月）
20問 約6.6週 約8週（2ヶ月）
30問 約4.3週 約5.7週（1.5ヶ月）
40問 約3.4週 約4.9週
50問 約2.7週 約4.1週（1ヶ月）

「確実に終わらせつつ生活リズム（TOEIC並行・就寝改善）を崩さない」バランスとして、1日30問・全体1.5ヶ月を採用した。

自分のための注意点

  • 計画づくりやTracker整備に時間を使い始めたら、それは先延ばしのサイン。計画は固めたら触らず、着手を優先する
  • 「間違いに印」と「OneNoteへの加筆」だけは省かない。これがフェーズ3の弱点2周目を成立させる土台になる
  • 量（ページ数・問題数）で進捗を管理する。時間で縛らないので、疲れた日は翌日に振り替えてよい
  • AZ-104は早く消すべき宿題なので、だらだら延ばさない。延ばすにしても全体2ヶ月まで

追記：1日30問は重すぎた（2026-06-18 見直し）

実際に問題集を解き始めて60問まで進めたところで止まった。原因は明確で、自分は「1問を理解できるまでどっぷり解く」性格だと分かったから。これ自体は資格勉強として正しいやり方だが、その丁寧さと1日30問は両立しない。普通の問題で5分、難しい問題で15分かかるため、30問は物理的に無理だった。

そこでペースと管理方法を変えた。

  • 1日30問 → 1日12問に減らす
  • 量だけでなく時間キャップを併用：開始から100分経過したら、その日は打ち切って翌日へ繰り越す（12問と100分の早く来た方で終了）
  • 1問をどっぷり理解するやり方は変えない（これは質の担保なので維持）
  • 全687問を完走する方針は維持。間引かず淡々と全部やる

見直し後の見積もり

残り627問 ÷ 12問/日 = 約53日。6/18開始で8/9頃に1周完了。お盆（8/15前後）受験には間に合うが、復習バッファは数日程度に縮む。そのため直前期は新規演習より「誤答マーク済みの問題・OneNote図解済み分野」の見直しを優先する。

運用ルール

  • 未達の日があっても翌日は通常の12問に戻す（借金を雪だるま式に増やさない）
  • 土日のどちらかを予備日にして、最大18問まで解ける日にする（平日の取りこぼし吸収）
  • 0問の日が出ても記録だけして責めない。数日のバッファがその保険

反省

6/3の計画で「30問・1.5ヶ月」を選んだのは、所要日数の表だけを見て自分の1問あたり実時間を計算に入れていなかったから。量ベースの管理は「1問が重い」性格と相性が悪く、時間キャップの併用で初めて崩れにくくなった。

まとめ

問題集の構成に合わせて「教科書先行・問題集集中型」に切り替え、1日30問ペースで全体1.5ヶ月を目安に進める計画にした。計画はこれ以上いじらず、教科書2-4 Azure CLIから着手する。
